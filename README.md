## Introduction 29.04
This is the first blog entry of my project. Implementation was done before the blog began so many subsequent entries will be written post-factum. Main idea of the project is to implement a technique called "Fast N-body simulation with GPU on CUDA" from GPU Gems 3 book on Unity engine. I will call this work the "original article" starting from here.(link) Core of the idea is multithreaded acceleration calculation done on the GPU which can be similarly implemented in Unity using a compute shader.
## Preliminal research and overall project outline 30.04 (actually done on 19.04)
To better understand what kind of souce code is needed for the project to work I spent some time researching the topic of compute shaders and thoroughly reading the original article. When loking at other implementations of updating postions of a large number of bodies in unity (namely this particular video by ___) I made important overarching decisions about the architecture of the final Unity project. Turned out that compute shader itself should essentially be a standalone function which will take in current system state and return updated positions and velocities. Compute shader is in turn called by the main control script, attached to some empty GameObject in Unity. Coordinates by themself are not enough to get an image, bodies need to be rendered by a shader. It would be possible to use a large amount of GameObjects but that is highly inneficient and so a specific shader needs to be created. This shader should take in only the positions and render appropriate meshes in those places. In total, system would consist of 3 parts: controller, shader and compute shader. (maybe a picture)
## Controller and shader 30.04 (20.04)
At first I decided to focus on getting the Unity to just render a cloud of bodies, without any movement. To do this a controller and the shader were needed. I decided to use VS Code for coding as it was already installed.  Creating a controller script was fairly straightforward, using aforementioned video as an example. For data storage an array of "body" structs were created, each including postions, velocities and mass. In contrast, writing a shader proved to be extremely difficult, as I have no experience with neither the engine nor the language or Unity shader specific methods. I have tried to modify the existing default shader code unity provides, asking questions from AI(Copilot in MS Edge), but to no avail. 

In the end I entrusted the whole shader to an AI agent, in this instance integrated Claude Haiku 4.5 agent inside VS Code to write it for me, based on my existing structs and general wishes.
